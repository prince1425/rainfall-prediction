Upload your NASA/POWER MERRA-2 CSV — monthly or daily — and this lab will run the dual-transform stacked ensemble end-to-end: feature engineering, chronological train/val/test split, six (or three) base learners trained on sqrt and log1p targets, a Ridge meta-learner, and full SHAP interpretability. § Dataset specification

Columns required for best prediction

This app is tuned for the exact parameter set used in the original Shillong-monthly and Guwahati-daily notebooks: 13 NASA/POWER MERRA-2 variables plus date/time columns. Your CSV must contain all of the columns below — if any are missing, the pipeline will stop with a clear error. Pull the file directly from the NASA/POWER portal with default settings to guarantee a match.

Target variable
Predict this
PRECTOTCORR Precipitation (bias-corrected) — the rainfall value the ensemble predicts. mm/day
Time columns
Required

DOY Day-of-year (1–366). Daily files only. Used to build the DATE column. 1–366

4 cols

3 cols

4 cols

1 col

§ 09 — Ridge Meta-Learner

How the stack combines its parts.

Linear weights the Ridge meta-learner assigned to each base model's raw-scale prediction. Large positive values = the meta-learner trusts that model strongly; small or negative values = the meta-learner is de-emphasizing it.
